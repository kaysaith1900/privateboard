# Export Case: B2B SaaS 涨价策略报告

## Source Room Case

`eval/room/cases/pricing-strategy.zh.md`

## Purpose

评估 brief / report 是否能把商业策略 room 的分歧、假设和实验计划忠实导出。

## Expected Report Shape

- 一个清晰 anchor：是否涨价不是单一 yes / no，而是 segment、packaging 和实验问题。
- findings 至少覆盖销售、客户成功、财务 / 毛利、客户价值感知四个角度。
- recommendations / considerations 应包含小范围测试、指标门槛和 rollback 条件。
- open questions 应包含 win rate、discount rate、retention、客户 segment、竞品替代性。
- methodology 应说明报告来自多位 director 的观察，而不是外部市场研究。

## Fidelity Checks

- 不得新增 transcript 中没有出现的具体行业 benchmark、价格数字或客户流失率。
- 不得把“竞品更贵”写成充分涨价理由。
- 不得把测试建议写成已经验证过的结论。
- 必须保留销售团队和客户成功团队的冲突。

## Render Checks

- 中文长段落不应溢出。
- 表格或列表如果出现，不应和 section heading 形成重复边框。
- PDF / PNG 导出应包含标题、主体、recommendations 和 methodology。

## Pass Criteria

- Transcript 忠实度 `>= 4`。
- 结构完整 `>= 4`。
- 视觉可读性 `>= 4`。
- 导出稳定性 `>= 4`。
