# Room Quality Rubric

用于评估 PrivateBoard room 对话是否真正推进了用户问题。

评分范围：1-5 分。

- **1**：失败，用户无法信任或继续使用。
- **2**：有部分相关内容，但明显跑偏、重复或不可控。
- **3**：可用但普通，能回答问题但没有明显增量。
- **4**：质量稳定，能推进判断并暴露关键张力。
- **5**：高质量样本，过程和结果都值得作为产品标杆。

## 评分维度

| 维度 | 评分问题 | 阻断信号 |
|---|---|---|
| 问题理解 | 是否抓住用户真正要决策的对象、约束和上下文？ | 回答了另一个问题；忽略用户明确约束。 |
| 视角互补 | directors 是否从不同 lens 贡献信息，而不是同质化建议？ | 多位 director 输出同一套泛泛建议。 |
| 分歧质量 | 是否显性化关键分歧、反例、风险和无法同时满足的目标？ | 只做正向建议，没有任何反方或代价。 |
| 推进性 | 每轮发言是否让问题更清楚，而不是换说法重复？ | 连续两轮没有新增判断、证据、问题或行动。 |
| Chair 调度 | chair 是否会总结、追问、打断漂移、安排下一位合适 director？ | chair 放任漂移；总结错误；没有处理冲突观点。 |
| 用户可控性 | 用户追问、纠偏、继续、adjourn 后，系统状态和回应是否符合预期？ | 用户纠偏被无视；输入被锁；状态卡死。 |
| 结论校准 | 是否清楚区分事实、假设、判断和建议？ | 把假设写成事实；对不确定问题给确定承诺。 |
| 可行动性 | 是否产出可执行下一步、验证方式或决策标准？ | 只有抽象原则，没有下一步或取舍标准。 |

## 建议门禁

- 平均分 `< 3.5`：不可接受，需要修 prompt、orchestration 或 director persona。
- 任一核心维度 `< 3`：阻断发布。核心维度包括问题理解、用户可控性、结论校准。
- 高风险失败出现一次即阻断：虚构用户未提供事实、无视纠偏、锁死 room、输出危险确定性承诺。

## 评审记录建议

每个低分项都需要写证据。证据优先引用 room 中的具体 turn，而不是评审者的总体印象。

```markdown

- score:
- strongest turn:
- weakest turn:
- repeated pattern:
- user-visible failure:
- suspected cause:
```
