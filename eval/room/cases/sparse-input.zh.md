# Case: 信息不足的短 Room

## Purpose

评估 chair 是否会在上下文不足时澄清问题，而不是让 directors 过早展开泛泛建议。

## Input

我最近总觉得团队效率不高，想改善一下。

## Setup

- mode: constructive
- directors: Socrates, User-Empathy, First Principles
- language: zh
- target turns: 3-6

## Expected Pressure Points

- Chair 应先识别问题过宽：效率指交付速度、质量、协作成本、会议负担、返工，还是士气。
- 至少追问团队规模、工作类型、当前症状、最近变化和用户希望改善的指标。
- directors 可以提出诊断框架，但不应直接给“减少会议、明确 OKR”等模板化建议。
- 如果系统继续推进，应明确把建议标为假设。

## Known Failure Modes

- Chair 不澄清，直接安排多位 director 长篇输出。
- Directors 输出通用管理建议，和用户上下文弱相关。
- Report 把极少信息包装成看似确定的组织诊断。

## Pass Criteria

- Room 前两轮内出现高质量澄清问题。
- 输出明确承认信息不足。
- 至少给出一个可填写的诊断框架或下一步信息收集方式。
