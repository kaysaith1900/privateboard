# Case: B2B SaaS 是否应该涨价

## Purpose

评估 room 在商业策略问题上的分歧构建、决策校准和行动建议质量。

## Input

我们做一个 B2B SaaS，主要客户是 50-500 人的团队，现在 ARR 大概 300 万美元，过去一年增长放慢了。销售团队建议把新客价格提高 25%，理由是竞品更贵，而且我们现在毛利压力比较大。但客户成功团队担心涨价会影响成交率和续约。我想知道我们到底该不该涨价，怎么判断？

## Setup

- mode: critique
- directors: Socrates, Value Investor, User-Empathy, First Principles
- language: zh
- target turns: 6-10

## Expected Pressure Points

- 区分新客涨价、老客涨价、套餐重构、折扣收紧和按价值分层。
- 要求拆开 win rate、sales cycle、gross retention、net retention、gross margin、discount rate。
- 识别销售团队和客户成功团队各自指标偏见。
- 提出小范围 cohort test 或 packaging experiment，而不是直接建议全面涨价。
- 明确涨价成功的前提：价值感知、替代品、ROI 证明、客户 segment。
- 讨论“不涨价”的机会成本和毛利压力，而不是只强调流失风险。

## Known Failure Modes

- directors 都重复“先做用户调研”。
- 只给涨价 / 不涨价结论，没有区分 segment。
- 把竞品更贵直接当作涨价理由。
- 忽略老客续约和新客成交是两个不同系统。
- 没有给出实验设计或决策门槛。

## Pass Criteria

- Room 至少形成两个互斥方案和一个推荐实验。
- Chair 能归纳销售视角与客户成功视角的真实冲突。
- 结论以假设和验证计划表达，而不是绝对判断。
