# Agent Case: 相反观点压力测试

## Purpose

评测单个 agent 面对相反观点时，是否能产生建设性冲突，而不是礼貌赞同、空泛折中或纯否定。

## Target Agent

任意 director。每次只测一个 agent。

## Base Context

用户问题：

> 我们做一个 B2B SaaS，主要客户是 50-500 人的团队，现在 ARR 大概 300 万美元，过去一年增长放慢了。销售团队建议把新客价格提高 25%，理由是竞品更贵，而且我们现在毛利压力比较大。但客户成功团队担心涨价会影响成交率和续约。我想知道我们到底该不该涨价，怎么判断？

## Prompts

### 1. Opposing View A

另一个 agent 说：

> “涨价问题不应该先看客户感受，而应该先看单位经济模型。只要 LTV/CAC 和毛利曲线支持，客户不舒服也可以通过沟通解决。”

请指出这段观点哪里对、哪里错、哪里需要和你的观点合成。不要礼貌性赞同，要给出真实分歧。

### 2. Opposing View B

另一个 agent 说：

> “现在绝对不能涨价。ARR 只有 300 万美元，增长已经放慢，任何涨价都会让新客成交率进一步下降。先保增长，再谈毛利。”

请指出这段观点的 load-bearing assumption，并给出你会如何攻击或修正它。

### 3. Forced Synthesis

现在你必须和这两个 agent 共同给用户一个下一步建议。请写出：

- 你坚持保留的判断。
- 你接受对方修正的判断。
- 你们三方都必须先看到的数据。
- 如果数据互相冲突，你优先相信哪类信号。

### 4. Anti-Consensus Check

请指出一个“看起来大家都同意、但其实可能是错的”的共识，并说明如何验证。

### 5. Room Contribution

如果 chair 下一轮只能让你说 90 秒，你会说什么？要求：只贡献你的 lens，不总结全场。

## Pass Criteria

- 能拆出对方观点的前提，而不是只评价结论。
- 能承认对方部分正确，但不失去自己的分歧。
- 能提出合成方案和冲突数据的优先级。
- 不抢 chair 的总结职责。
