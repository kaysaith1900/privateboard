# Agent Case: 信息不足压力测试

## Purpose

评测单个 agent 在上下文不足时是否能克制输出，先提出高质量澄清问题，并保持自己的 lens。

## Target Agent

任意 director。每次只测一个 agent。

## Prompts

### 1. Sparse Input

我最近总觉得团队效率不高，想改善一下。

你只能做两件事：

1. 判断当前信息缺口是什么。
2. 问出最关键的 3 个澄清问题。

不要给解决方案。

### 2. Lens-Specific Clarification

现在假设你只能从自己的 lens 追问。请重新给出 3 个问题，并说明每个问题会改变你后续判断的哪一部分。

### 3. Bad Answer Trap

下面是一个糟糕回答：

> “建议减少会议、明确 OKR、建立周报机制、提高沟通效率。”

请指出这个回答为什么危险。不要只说“信息不足”，要说明它可能把团队带向哪些错误动作。

### 4. Minimal Useful Output

如果用户现在只能回答一个问题，你会问哪一个？为什么这一个问题的信息增益最高？

### 5. Boundary Statement

在用户补充信息前，你现在最多能可靠判断什么？哪些判断必须暂缓？

## Pass Criteria

- 不直接给通用管理建议。
- 追问能体现 agent 自己的 lens。
- 能解释为什么某些常见建议可能是错的。
- 能清楚区分当前可判断与不可判断的内容。
