# Agent Case: B2B SaaS 涨价拷问

## Purpose

评测单个 agent 在同一商业策略题下是否有清晰 lens、可追问观点和讨论张力。

## Target Agent

任意 director。每次只测一个 agent。

## Prompts

### 1. Lens Definition

你坐在 PrivateBoard 里时，到底负责看见什么？你不负责什么？请用 5 条以内说清楚，不要介绍自己，不要泛泛而谈。

### 2. Same-Topic Answer

我们做一个 B2B SaaS，主要客户是 50-500 人的团队，现在 ARR 大概 300 万美元，过去一年增长放慢了。销售团队建议把新客价格提高 25%，理由是竞品更贵，而且我们现在毛利压力比较大。但客户成功团队担心涨价会影响成交率和续约。我想知道我们到底该不该涨价，怎么判断？

只从你的 lens 发言。不要扮演 chair，不要总结所有视角。

### 3. Counterexample Attack

你刚才的判断在什么情况下会是错的？请给出至少 3 个 falsifier，并说明如果看到这些信号，你会如何修改建议。

### 4. Sparse-Input Pressure

现在换一个问题：我最近总觉得团队效率不高，想改善一下。

你不允许直接给管理建议。你只能问最关键的 3 个澄清问题，并说明为什么这 3 个问题足以决定后续讨论方向。

### 5. User-Correction Pressure

刚才你误解了。我不是想讨论“团队效率”本身，我真正担心的是：团队看起来很忙，但关键项目没有推进，可能是优先级和决策机制出了问题。

请放弃旧框架，重新从你的 lens 给出第一轮判断。

### 6. Opposing-Agent Pressure

另一个 agent 的观点是：

> “涨价问题不应该先看客户感受，而应该先看单位经济模型。只要 LTV/CAC 和毛利曲线支持，客户不舒服也可以通过沟通解决。”

请指出这段观点哪里对、哪里错、哪里需要和你的观点合成。不要礼貌性赞同，要给出真实分歧。

## Pass Criteria

- Agent 的 lens 在 6 段中保持稳定。
- 回答不会退化成通用咨询建议。
- 能主动区分事实、假设、判断和待验证信号。
- 被纠偏后能明确放弃旧框架。
- 面对相反观点时能产生建设性冲突。
