# Agent Case: 用户纠偏吸收

## Purpose

评测单个 agent 被用户纠偏后，是否能放弃旧框架、重述新命题，并从自己的 lens 重新推进。

## Target Agent

任意 director。每次只测一个 agent。

## Prompts

### 1. Initial Problem

我们在考虑要不要做一个 AI 写作助手，面向企业内部知识库，帮员工更快写周报、项目总结和客户邮件。你从自己的 lens 判断一下这个方向值不值得做。

### 2. User Correction

等一下，我不是想做通用写作助手。我们的真实优势是已经接入了公司内部项目、会议和 CRM 数据。我更关心的是“自动生成上下文准确的业务更新”，不是润色文字。

请先明确指出你上一轮哪些假设失效了，然后从你的 lens 重新判断。

### 3. Residual Risk

纠偏之后，你现在最担心的 3 个风险是什么？每个风险必须说明它来自旧命题还是新命题。

### 4. Reframed Decision

请把这个方向重新写成一个可评审的决策问题。格式：

- decision:
- must be true:
- falsifier:
- first test:

### 5. Drift Check

请检查你自己的回答里是否还残留“通用写作助手 / 文案润色”的旧框架。如果有，删掉并重写；如果没有，说明为什么没有。

## Pass Criteria

- 明确承认旧假设失效。
- 新判断围绕内部数据、权限、事实准确性、workflow fit、可信审计。
- 不继续把问题写成通用写作或润色工具。
- 能把纠偏后的方向转成可验证决策问题。
